nsact SQL с использованием генератора парсеров ANTLR4.
Использовать рантайм для C++.
Для парсера реализовать CLI.

Информация, которая опционально должна возвращаться:
- корректный или некорректный запрос;
- время разбора;
- время второго разбора (ANTLR использует кэш поэтому второй разбор будет быстрее)
- перечень токенов (лексем);
- дерево разбора в форматах string и JSON;
- перечень всех столбцов, баз данных, схем и таблиц из запроса: например, для запроса 
"SELECT c1,c2 from tab1" должно быть получено что-то вроде
{
  "columns": ["c1", "c2"],
  "tables": ["tab1"]
}
Код должен быть размещен на github, bitbucket и т.п.
Проект должен быть документирован. Должно быть четко описано: что сделать, чтобы его запустить, как с ним работать и т.п.
Грамматика для tsql - https://github.com/antlr/grammars-v4/tree/master/tsql
